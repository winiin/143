# My Way — бэкенд

Реальный сервер для приложения: регистрация/логин с bcrypt, JWT-токены,
синхронизация данных в SQLite, и тестовый платёжный флоу.

## Запуск локально

```bash
cd server
npm install
npm start
```

Откройте **http://localhost:3000** — сервер отдаёт и фронтенд (`../index.html`),
и API (`/api/...`), так что всё работает с одного адреса без CORS-проблем.

Перед деплоем в прод обязательно задайте свой секрет:

```bash
JWT_SECRET="ваш-случайный-секрет" npm start
```

## Что реально работает

- **Регистрация/логин** — пароль хэшируется через `bcrypt` (12 раундов) на
  сервере, никогда не хранится и не сравнивается в открытом виде.
- **JWT-токен** — выдаётся на 30 дней, проверяется на каждый запрос к
  защищённым эндпоинтам.
- **Синхронизация данных** — `/api/data` (GET/PUT), хранится в SQLite
  (`myway.db`, создаётся автоматически).
- **Тестовые платежи** — `/api/payment/create` создаёт заявку со статусом
  `pending`, `/api/payment/test-confirm` подтверждает её и выдаёт premium.
  Статус реально хранится в базе, а не только в браузере.

## Что НЕ является настоящим (честно)

`test-confirm` — это имитация ответа платёжного шлюза, а не реальное
списание денег. Чтобы подключить настоящий платёж (Kaspi Pay, Stripe и
т.п.), нужно:

1. Завести мерчант-аккаунт у провайдера и получить ключи.
2. Заменить содержимое `/api/payment/test-confirm` на реальный вызов
   API провайдера (или принимать вебхук от него на отдельном роуте).
3. Всё остальное — создание заявки, хранение статуса, выдача premium —
   останется без изменений, потому что это уже сделано правильно.

## Если бэкенд не запущен

Фронтенд (`app.js`) при недоступности `/api/*` автоматически откатывается
на локальный режим (localStorage) — регистрация/логин работают прямо в
браузере, а оплата показывает экран "оплата скоро появится" вместо
реального теста. Это сделано специально, чтобы демо не ломалось, если
кто-то откроет `index.html` напрямую без запущенного сервера.

## Структура БД (SQLite, файл `myway.db`)

- `users` — email, имя, хэш пароля, статус premium и дата его окончания
- `user_data` — JSON с данными приложения (транзакции, цели, карты и т.д.)
- `payments` — история платежей (план, сумма, статус, даты)
